BLT-Claw
OpenClaw Bot on Your VPS

One script. One playbook. A fully configured OpenClaw AI assistant connected to your OWASP BLT Slack workspace — running as a hardened systemd daemon on your own infrastructure.

Get Started View Source

What BLT-Claw Gives You

Everything you need to run a production-grade AI assistant bot on a bare VPS — no cloud subscriptions, no lock-in.

One-Line Install

setup.sh handles Node.js, OpenClaw, systemd service, and Slack channel config end-to-end — from a fresh Ubuntu/Debian/Rocky VPS.

Ansible Playbook

Idempotent Ansible role with templated config, vault-ready secrets, and automatic service management — repeatable across fleets.

Slack Integration

Socket Mode Slack bot using bot + app tokens. DM pairing policy keeps the channel secure — unknown senders receive a pairing code before the bot responds.

Hardened by Default

Dedicated system user, NoNewPrivileges, PrivateTmp, ProtectSystem, and config files locked to 0600.

Any AI Model

OpenAI, Anthropic, local models — configure your preferred provider and model via a single variable. Swap models without touching the service.

Auto-Restart

systemd Restart=on-failure keeps the gateway running after crashes. Enable on boot so the bot survives VPS reboots automatically.

Quick Start

From a fresh Ubuntu/Debian/Rocky VPS to a running Slack bot in minutes.

  1. 1

    Create a Slack App

    Go to api.slack.com/apps, create a new app from scratch, enable Socket Mode, and add the following Bot Token Scopes under OAuth & Permissions:

    app_mentions:read channels:history chat:write im:history im:read im:write

    Note your Bot Token (xoxb-…) and generate an App-Level Token with connections:write scope (xapp-…).

  2. 2

    SSH into your VPS and run setup.sh

    bash 
    # RECOMMENDED — download, review, then run
curl -fsSL https://raw.githubusercontent.com/OWASP-BLT/BLT-Claw/main/setup.sh -o setup.sh
less setup.sh   # inspect before running!
sudo bash setup.sh

# Alternative — clone the repo and run
git clone https://github.com/OWASP-BLT/BLT-Claw.git
cd BLT-Claw
sudo bash setup.sh

    The script will prompt you for your Slack tokens and AI model API key if they are not already set as environment variables.

  3. 3

    Pre-set credentials (optional)

    bash 
    export OPENCLAW_SLACK_BOT_TOKEN="xoxb-your-bot-token"
export OPENCLAW_SLACK_APP_TOKEN="xapp-your-app-token"
export OPENCLAW_MODEL_API_KEY="sk-your-openai-key"
export OPENCLAW_MODEL_PROVIDER="openai"   # optional, default: openai
export OPENCLAW_MODEL_NAME="gpt-4o"       # optional, default: gpt-4o

sudo -E bash setup.sh
  4. 4

    Message the bot and approve the pairing code

    Invite the bot to a Slack channel, send it a DM, and it will reply with a pairing code. Approve it from the server:

    bash 
    openclaw pairing approve slack <CODE>

Ansible Deployment

For fleet deployments or repeatable infrastructure-as-code setups, use the included Ansible role.

Prerequisites

  • Ansible ≥2.14 on your control node (pip install ansible)
  • community.general collection for the npm module
  • SSH key-based access to your VPS
bash 
pip install ansible
ansible-galaxy collection install community.general

Edit the inventory

Update ansible/inventory.yml with your VPS IP or hostname.

YAML 
all:
  children:
    vps_servers:
      hosts:
        openclaw-vps:
          ansible_host: "203.0.113.10"   # ← your VPS IP
          ansible_user: root

Run the playbook

bash 
ansible-playbook -i ansible/inventory.yml ansible/setup.yml \
  --extra-vars "openclaw_slack_bot_token=xoxb-... \
                openclaw_slack_app_token=xapp-... \
                openclaw_model_api_key=sk-..."

For production use, store secrets in an ansible-vault encrypted file and pass it with --vault-password-file.

Day-to-Day Operations

Common commands for managing the OpenClaw gateway.

Task Command
Check service status systemctl status openclaw
Follow live logs journalctl -u openclaw -f
Restart the gateway systemctl restart openclaw
Run health check openclaw doctor
Approve pairing code openclaw pairing approve slack <CODE>
Update OpenClaw npm install -g openclaw@latest && systemctl restart openclaw
View config file cat /opt/openclaw/home/.openclaw/config.json
Send a test message openclaw agent --message "ping"

Configuration Reference

The config file lives at /opt/openclaw/home/.openclaw/config.json and is readable only by the openclaw system user. Edit it then systemctl restart openclaw.

config.json 
{
  "gateway": {
    "port": 18789,
    "host": "127.0.0.1"
  },
  "models": [
    {
      "provider": "openai",
      "name": "gpt-4o",
      "apiKey": "sk-…"
    }
  ],
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "xoxb-…",
      "appToken": "xapp-…",
      "dmPolicy": "pairing",
      "allowFrom": []
    }
  }
}
